This application contains intentional 403-forbidden bypass vulnerabilities for educational purposes:

1. SQL Injection in Admin Check

The is_admin() function uses vulnerable SQL queries that can be exploited.

query = f"SELECT is_admin FROM users WHERE id = {user_id}"
2. Header-Based Bypass

API endpoints check for specific headers that can be easily spoofed.

if request.headers.get('X-Admin-Key') != 'admin_secret_key_2024':
3. User Agent Bypass

Some endpoints check user agent strings for admin access.

if 'admin' not in user_agent.lower():
4. IP Address Bypass

System status endpoint only checks client IP addresses.

if client_ip not in ['127.0.0.1', '::1']:
Bypass Techniques:
  • SQL Injection: Use ' OR 1=1 -- in user ID parameters
  • Header Manipulation: Add X-Admin-Key: admin_secret_key_2024
  • User Agent Spoofing: Set user agent to contain "admin"
  • IP Spoofing: Use localhost or proxy through 127.0.0.1
  • Session Manipulation: Modify session cookies to set admin privileges
